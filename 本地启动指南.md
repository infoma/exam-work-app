# 本地启动指南（Windows）

## 环境要求

- **Node.js** >= 18（必须）
  - 下载地址：https://nodejs.org/  选 LTS 版本
  - 安装时勾选 "Add to PATH"
- **Git**（必须）
  - 下载地址：https://git-scm.com/download/win
- **VS Code 或 Trae IDE**（推荐，方便查看代码）

---

## 启动步骤

### 第 1 步：克隆代码

打开 PowerShell 或 CMD，执行：

```powershell
cd C:\Users\Administrator
git clone https://github.com/infoma/exam-work-app.git
```

如果提示需要输入用户名密码，说明仓库是私有的，需要把 GitHub Token 加到 URL 里：

```powershell
git clone https://ghp_你的Token@github.com/infoma/exam-work-app.git
```

### 第 2 步：进入后端目录

```powershell
cd exam-work-app\server
```

### 第 3 步：安装依赖

```powershell
npm install
```

这个过程可能需要几分钟，看到 `added xxx packages` 就表示成功了。

### 第 4 步：启动服务

```powershell
npm run start:dev
```

看到以下信息表示启动成功：

```
[Nest] xxx  - xx/xx/xxxx, x:xx:xx xx     LOG [NestApplication] Nest application successfully started
```

### 第 5 步：验证服务

**不要关闭刚才的 PowerShell 窗口**（服务在运行中），另开一个 PowerShell 窗口：

```powershell
curl http://localhost:3000/
```

如果返回 JSON 数据，说明服务启动成功！

## 测试新模块

服务启动后，另开 PowerShell 窗口执行：

### 注册用户

```powershell
curl -X POST http://localhost:3000/api/auth/register `
  -H "Content-Type: application/json" `
  -d '{"username":"admin","password":"123456","name":"管理员"}'
```

### 登录

```powershell
curl -X POST http://localhost:3000/api/auth/login `
  -H "Content-Type: application/json" `
  -d '{"username":"admin","password":"123456"}'
```

复制返回的 `accessToken` 值（去掉引号）。

### 测试生源学校

```powershell
$token = "你的accessToken"

curl -X POST http://localhost:3000/api/source-schools `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer $token" `
  -d '{"name":"北京市第一中学","code":"BJ001","schoolType":"高中","province":"北京市","city":"北京市","district":"东城区","contactPerson":"张老师","contactPhone":"010-12345678","studentCount":1200}'
```

### 查看学校列表

```powershell
curl http://localhost:3000/api/source-schools `
  -H "Authorization: Bearer $token"
```

## 停止服务

回到刚才运行 `npm run start:dev` 的 PowerShell 窗口，按 `Ctrl + C`，然后输入 `Y` 确认。

---

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| `npm 不是内部或外部命令` | Node.js 没装好或 PATH 没配置，重装 Node.js 并勾选 "Add to PATH" |
| `git clone 失败` | 检查网络，或把 Token 写到 URL 里 |
| `npm install 卡住` | 换淘宝镜像：`npm config set registry https://registry.npmmirror.com` |
| `端口 3000 被占用` | 关闭其他占用 3000 端口的程序，或修改 `.env` 文件里的 PORT |
| `curl 命令报错` | Windows 的 curl 和 Linux 不同，建议装 Git Bash 或在 CMD 里用 `Invoke-WebRequest` |

## 用浏览器测试（更直观）

服务启动后，浏览器打开：

- http://localhost:3000/  — 看服务状态
- http://localhost:3000/api  — 看 API 接口列表

如果装了 Swagger，可能还有：
- http://localhost:3000/api/docs  — Swagger 文档（如果项目配置了）
